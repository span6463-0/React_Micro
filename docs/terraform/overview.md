# Terraform — AWS Infrastructure

All AWS resources are managed with Terraform. The project uses reusable modules and environment-specific variable files.

**Location**: `Terraform/`  
**Terraform Version**: >= 1.0  
**AWS Provider Version**: ~> 5.0

---

## Directory Structure

```
Terraform/
├── main.tf              Provider + backend config (S3 + DynamoDB state)
├── infrastructure.tf    Root module — wires all modules together
├── variables.tf         Input variable definitions
├── outputs.tf           Exported resource IDs and URLs
├── environments/
│   ├── dev/terraform.tfvars
│   └── prod/terraform.tfvars
└── modules/
    ├── vpc/             VPC, subnets, NAT gateways, route tables
    ├── ecr/             ECR repositories for all services
    ├── ecs/             ECS Fargate cluster, IAM roles, security groups
    ├── rds/             RDS PostgreSQL instances + Secrets Manager
    └── alb/             Application Load Balancer + HTTPS listener
```

## Modules

### VPC Module (`modules/vpc/`)

Creates a production-ready VPC with:

- **Public subnets** — ALB, NAT Gateway EIPs
- **Private subnets** — ECS Fargate tasks
- **Database subnets** — RDS instances (no internet access)
- **NAT Gateways** — one per AZ for HA egress from private subnets
- **Route tables** — separate tables per subnet tier
- **DB Subnet Group** — used by all RDS modules

**Key outputs**: `vpc_id`, `public_subnet_ids`, `private_subnet_ids`, `db_subnet_group_name`

---

### ECR Module (`modules/ecr/`)

Creates ECR repositories for all 8 services:

- `react-micro/shell`
- `react-micro/landing-page`
- `react-micro/dashboard`
- `react-micro/item-management`
- `react-micro/bff-gateway`
- `react-micro/auth-api`
- `react-micro/user-api`
- `react-micro/item-api`

Features:

- Image scan on push enabled
- AES256 encryption
- Lifecycle policy: retain last 10 images per repo

---

### ECS Module (`modules/ecs/`)

Creates the ECS Fargate cluster with:

- Container Insights enabled
- FARGATE + FARGATE_SPOT capacity providers
- IAM execution role (pull from ECR, write to CloudWatch)
- IAM task role (for application-level AWS access)
- Security group allowing intra-VPC communication
- CloudWatch log group: `/ecs/{project}-{env}` (30-day retention)

---

### RDS Module (`modules/rds/`)

Creates a PostgreSQL 15 RDS instance per service. Used three times (auth, user, item):

- Master password auto-generated by `random_password` and stored in **AWS Secrets Manager**
- **Multi-AZ** enabled in `prod`, disabled in `dev`
- **Deletion protection** enabled in `prod`
- Enhanced monitoring (60s interval) with IAM role
- Performance Insights enabled
- gp3 storage with auto-scaling (2× allocated)

**Secrets Manager secret name pattern**: `{project}-{env}-{service}-db-password`

---

### ALB Module (`modules/alb/`)

Creates an internet-facing Application Load Balancer:

- HTTP listener redirects to HTTPS when `certificate_arn` is provided
- HTTPS listener uses TLS 1.3 policy
- Deletion protection enabled in `prod`
- Default target group with `/health` health check

## Environment Configurations

### Development (`environments/dev/terraform.tfvars`)

```hcl
environment        = "dev"
db_instance_class  = "db.t3.micro"
ecs_task_cpu       = 256
ecs_task_memory    = 512
```

### Production (`environments/prod/terraform.tfvars`)

```hcl
environment        = "prod"
availability_zones = ["us-east-1a", "us-east-1b", "us-east-1c"]
db_instance_class  = "db.t3.medium"
ecs_task_cpu       = 512
ecs_task_memory    = 1024
```

## Remote State

State is stored in S3 with DynamoDB locking:

```hcl
backend "s3" {
  bucket         = "react-micro-terraform-state"
  key            = "terraform.tfstate"
  region         = "us-east-1"
  dynamodb_table = "react-micro-terraform-lock"
  encrypt        = true
}
```

> **Before first run**: Create the S3 bucket and DynamoDB table manually or with a bootstrap script.

## Deployment Workflow

### First-Time Setup

```bash
cd Terraform

# Initialize (download providers, configure backend)
terraform init

# Review what will be created
terraform plan -var-file="environments/dev/terraform.tfvars"

# Apply infrastructure
terraform apply -var-file="environments/dev/terraform.tfvars"
```

### Updating Infrastructure

```bash
# Preview changes
terraform plan -var-file="environments/dev/terraform.tfvars" -out=tfplan

# Apply after review
terraform apply tfplan
```

### Destroying Dev Environment

```bash
terraform destroy -var-file="environments/dev/terraform.tfvars"
```

> **Warning**: Never run `destroy` against production without explicit approval.

## Tagging Strategy

All resources are tagged with:

| Tag           | Value                                          |
| ------------- | ---------------------------------------------- |
| `Project`     | `react-micro`                                  |
| `Environment` | `dev` / `staging` / `prod`                     |
| `ManagedBy`   | `terraform`                                    |
| `Service`     | Service-specific (e.g., `auth`, `bff-gateway`) |

---

## Security Considerations

- RDS instances are in **database subnets** with no public access
- ECS tasks are in **private subnets**; only ALB is public-facing
- All secrets stored in **AWS Secrets Manager** (never in code)
- Encryption at rest enabled on all RDS instances and ECR repos
- ALB enforces HTTPS (TLS 1.3) in production
